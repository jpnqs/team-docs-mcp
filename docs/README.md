# Team Knowledge Base - README

This folder contains the team's documentation, best practices, guidelines, and knowledge base. All documents are indexed and searchable through the Team Docs MCP server.

## Documentation Structure

- **[coding-standards.md](./coding-standards.md)** - Code style, naming conventions, and best practices
- **[api-conventions.md](./api-conventions.md)** - REST API design guidelines and standards
- **[deployment-guide.md](./deployment-guide.md)** - Deployment procedures and environment management
- **[development-setup.md](./development-setup.md)** - Setting up the local development environment

## Adding Documentation

1. Create markdown files in this directory
2. Use clear, descriptive filenames
3. Include relevant examples and code snippets
4. Link to related documents
5. Restart the MCP server to index new documents

## Search Tips

When using the semantic search:

- Ask natural language questions
- Be specific about what you're looking for
- Try rephrasing if results aren't relevant
- Top results (highest similarity scores) are most relevant

## Example Queries

Try these queries with the `search-documentation` tool:

- "How do we handle authentication in our APIs?"
- "What are the naming conventions for TypeScript?"
- "How to deploy to production?"
- "What's the database migration process?"
- "Error handling best practices"
- "How to run tests?"

## Maintenance

- Keep documentation up to date
- Remove outdated information
- Add new team decisions and standards
- Review and update quarterly
